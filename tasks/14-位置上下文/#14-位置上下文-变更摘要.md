## 三、后端 LocationService 设计

### 请求流程

```
GET /api/ai/location-context?lat=31.23&lng=121.47
         │
         ▼
   ┌─ 检查缓存 ──── 命中(≤5min) → 直接返回
   │   未命中
   ▼
   Nominatim 逆地理编码 ── 失败 → 降级为坐标字符串
   │   成功
   ▼
   GPT-4o-mini 生成描述 + moods ── 失败 → 模板降级
   │
   ▼
   写入缓存 → 返回 LocationContextResponse
```

### 缓存策略
- Key: `(round(lat,4), round(lng,4))` ≈ 11m 精度
- TTL: 300 秒（5 分钟）
- 存储: 内存 dict（服务生命周期内有效）

### 降级策略
| 场景 | 行为 |
|------|------|
| Nominatim 请求失败 | `name` 降级为 `"31.2304, 121.4737"` 坐标字符串 |
| Nominatim 返回空 display_name | 同上 |
| OPENAI_API_KEY 未配置 | `description` 使用模板文案，`suggested_moods` 固定为 `["怀旧","温暖","希望"]` |
| GPT 调用异常/超时 | 同上 |

### 返回格式（与 Pydantic 模型一致）

```json
{
  "name": "延安东路, 均乐社区, 南京东路街道",
  "description": "在延安东路附近，这里有来自周围的时空胶囊...",
  "nearby_capsule_count": 0,
  "suggested_moods": ["怀旧", "温暖", "希望"]
}
```

## 四、前端改动

### HomePage.tsx 的变化

1. **新增 location context 请求**：GPS 坐标变化超过 ~100m 时自动调用 `aiApi.getLocationContext(lat, lng)`，结果存入 `useCapsuleStore.locationContext`

2. **底部面板改为独立数据源**：原来从 `nearby?.location_context` 读取（随 nearby 接口返回），现在改用独立的 `/api/ai/location-context` 端点数据 `locationContext`

3. **新增 suggested_moods 展示**：底部面板会显示 AI 推荐的心情标签胶囊（与情绪标签同款样式）

> ⚠️ **注意**：`nearby_capsule_count` 目前始终为 0。如果后续需要让 location-context 端点感知附近胶囊数量，可以在 `capsules.py` 的 `/nearby` 路由中注入真实数据，或者前端拿到 nearby 响应后手动更新。

## 六、如何测试

### 后端

```bash
cd backend
uvicorn app.main:app --reload --port 8000

# 测试端点
curl "http://localhost:8000/api/ai/location-context?lat=31.2304&lng=121.4737"
```

如果有 OPENAI_API_KEY，把 key 写入 `backend/.env`：
```
OPENAI_API_KEY=sk-xxx
```

重启后即可看到 GPT 生成的真实描述和心情标签。

### 前端

```bash
cd frontend
npm run dev
```

打开首页，底部面板会显示当前位置的 AI 上下文信息（名称、描述、推荐心情标签）。
